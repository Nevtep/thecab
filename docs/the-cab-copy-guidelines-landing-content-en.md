# The Cab — App Copy Guidelines & Landing Content v1.0

## Document role

This document defines the textual content and copy direction for **The Cab**.

It is meant for:

- product design;
- landing page implementation;
- connected app empty states;
- onboarding / analysis states;
- i18n resource generation;
- AI-assisted coding.

This document does **not** replace the product specification or brand specification. It translates the existing product and brand direction into usable product copy.

The canonical language for product copy is **English**. Spanish is the second supported language and should be translated from the English source while preserving tone, precision, and technical meaning.

---

# 1. Copy direction

## 1.1 Voice

The Cab should sound:

- precise;
- technical;
- confident;
- crypto-native;
- premium;
- disciplined;
- operational;
- analytical;
- calm under complexity.

The Cab should not sound:

- meme-driven;
- casino-like;
- retail-trading oriented;
- exaggerated;
- childish;
- lifestyle-first;
- generic SaaS;
- overly casual.

The Cab is crypto-native, but not hype-driven.

Use the language of DeFi analytics, capital flows, protocols, pools, deposits, strategies, rewards, governance, wallet activity, and attribution.

Avoid language that sounds like meme-coin culture, speculative trading bait, or “instant alpha” marketing.

## 1.2 Core positioning

**The Cab is a control tower for on-chain capital.**

It helps Aerodrome users understand:

- where capital is deployed;
- how exposure evolved;
- which pools matter;
- which deposits and strategies performed;
- where rewards came from;
- what happened in wallet activity;
- which data is complete, partial, or inferred.

## 1.3 Copy principles

### Be specific

Prefer:

```txt
Analyze pool exposure, deposits, rewards, and governance activity.
```

Avoid:

```txt
Understand everything about your DeFi journey.
```

### Be operational

Prefer:

```txt
Run historical analysis to unlock pool, deposit, strategy, reward, and governance views.
```

Avoid:

```txt
Unlock the future of your portfolio.
```

### Be crypto-native but not noisy

Prefer:

```txt
Track deployed capital, residual balances, rewards, and strategy exposure.
```

Avoid:

```txt
Find alpha and ape smarter.
```

### Make uncertainty visible

Prefer:

```txt
Strategy coverage: share-level.
```

Avoid:

```txt
Fully decoded.
```

unless the system actually has complete coverage.

### Separate analysis from execution

The Cab is not a trading or execution terminal.

Prefer:

```txt
Monitor and reconstruct historical capital flow.
```

Avoid:

```txt
Trade better.
```

- Control tower for on-chain capital

## 2.2 Avoided phrases

Avoid:

- Moon
- Ape
- Degen
- Insane yields
- Guaranteed returns
- Find alpha instantly
- Profit machine
- Beat the market
- Autopilot profits
- Risk-free
- Always accurate
- Fully decoded, unless actually true
- Real-time, unless technically true

## 2.3 Protocol terms that should remain untranslated or preserved

These terms should generally remain as protocol/product terms across languages:

- Aerodrome
- Mellow
- AERO
- veAERO
- Base
- Pool
- Gauge
- Router
- StakingRewards
- lpWrapper
- Wallet
- DeFi
- Strategy
- Deposit
- Governance

In Spanish translations, terms like “Pool”, “Strategy”, and “Deposit” may be translated only if the product team decides to localize DeFi terminology. Product v1 should prioritize clarity for crypto users.
