# The Cab — Guía de Copy y Contenido en Español v1.0

## Rol del documento

Este documento define la versión en español del contenido textual para **The Cab**.

No es una traducción literal del documento en inglés. Es una **adaptación cultural y de producto** para usuarios hispanohablantes, especialmente usuarios cripto/DeFi familiarizados con términos como wallet, pool, rewards, governance, strategy y Base.

Este documento está pensado para:

- diseño de producto;
- implementación de la landing;
- textos de estados vacíos;
- estados bloqueados;
- estados de análisis;
- recursos i18n en español;
- desarrollo asistido por IA.

El idioma canónico del producto sigue siendo **inglés**, pero la versión en español debe sentirse natural, técnica y precisa. No debe sonar como traducción automática.

# 1. Dirección de copy en español

## 1.1 Voz

The Cab en español debe sonar:

- preciso;
- técnico;
- sobrio;
- crypto-native;
- premium;
- operativo;
- analítico;
- confiable;
- claro sin ser básico.

No debe sonar:

- exagerado;
- vendehumo;
- meme;
- casino cripto;
- influencer financiero;
- trading retail agresivo;
- demasiado corporativo;
- genérico SaaS;
- infantil o “gamificado”.

The Cab es un producto cripto, pero no se comunica desde el hype.

Debe hablar el lenguaje de:

- DeFi;
- capital on-chain;
- pools;
- depósitos;
- estrategias;
- rewards;
- governance;
- wallet activity;
- atribución;
- cobertura;
- análisis histórico.

Debe evitar lenguaje tipo:

- “a la luna”;
- “modo degen”;
- “ganancias aseguradas”;
- “encontrá alpha instantáneamente”;
- “multiplicá tu portfolio”;
- “yields increíbles”.

## 1.2 Posicionamiento central

**The Cab es una torre de control para tu capital on-chain.**

Ayuda a usuarios de Aerodrome a entender:

- dónde está desplegado su capital;
- cómo evolucionó su exposición;
- en qué pools participó;
- cómo rindieron sus depósitos y estrategias;
- de dónde vinieron sus rewards;
- qué pasó en su actividad de wallet;
- qué datos están completos, parciales o inferidos.

# 2. Lenguaje clave del producto

## 2.1 Frases principales

Usar estas frases de manera consistente:

- Torre de control para capital on-chain
- Cabina de comando del portfolio
- Analytics para portfolios en Aerodrome
- Flujo histórico de capital
- Exposición por pool
- Depósitos manuales
- Estrategias automatizadas
- Exposición en estrategias Mellow
- Atribución de rewards
- Retornos de governance
- Reconstrucción de actividad de wallet
- Balances residuales
- Detección de rebalances
- Estado de cobertura
- Accounting a nivel de shares

## 2.2 Términos a evitar

Evitar:

- A la luna
- Ape
- Degen, salvo en contextos muy deliberados y nunca en la UI core
- Yields increíbles
- Ganancias garantizadas
- Encontrá alpha instantáneamente
- Máquina de ganancias
- Ganale al mercado
- Profit automático
- Sin riesgo
- Siempre exacto
- Totalmente decodificado, salvo que sea técnicamente cierto
- En tiempo real, salvo que sea técnicamente cierto

## 2.3 Términos que conviene preservar

Estos términos pueden mantenerse en inglés porque son naturales para usuarios DeFi:

- Wallet
- Pool
- Strategy / Estrategia
- Rewards / Recompensas
- Governance
- DeFi
- Gauge
- Router
- StakingRewards
- lpWrapper
- Base
- Aerodrome
- Mellow
- AERO
- veAERO

Recomendación de producto para español:

- Usar **wallet**, no “billetera”, en la UI principal.
- Usar **pool**, no “piscina”.
- Usar **rewards** si el público es cripto; “recompensas” puede usarse en textos explicativos.
- Usar **governance** en navegación o **Gobernanza** si se prefiere una UI más localizada. Recomendación: `Governance` para mantener precisión DeFi.
- Usar **estrategia** para “strategy” en texto largo, pero `Strategies` puede mantenerse si se busca consistencia con protocolo/producto.

## 3.3 Sección problema

### Eyebrow

```txt
Los datos están on-chain. La lectura no siempre es clara.
```

### Título

```txt
El historial de una wallet no está diseñado para explicar performance.
```

### Cuerpo

```txt
La actividad en Aerodrome puede pasar por pools, gauges, depósitos manuales, estrategias Mellow, claims, votos, swaps, retiros y balances residuales. Un historial de wallet muestra transacciones. The Cab reconstruye la imagen operativa detrás de esas transacciones.
```

### Cards de problema

#### Card 1

Título:

```txt
La exposición por pool se fragmenta
```

Cuerpo:

```txt
El capital puede moverse entre depósitos, balances residuales, rebalances y nuevas posiciones. The Cab mantiene visible la exposición por pool en lugar de tratar cada retiro como una salida definitiva.
```

#### Card 2

Título:

```txt
Las estrategias no son depósitos manuales
```

Cuerpo:

```txt
Las estrategias Mellow usan wrappers, shares, contratos de rewards y su propio modelo de cobertura. The Cab separa la exposición automatizada de los depósitos manuales en Aerodrome.
```

#### Card 3

Título:

```txt
Los rewards necesitan atribución
```

Cuerpo:

```txt
Claims, fees, bribes y rewards de governance son útiles cuando pueden vincularse a pools, depósitos, estrategias o votos.
```

#### Card 4

Título:

```txt
La actividad necesita interpretación
```

Cuerpo:

```txt
Un swap puede ser una operación normal, un rebalance, una liquidación o parte de un flujo de atribución residual. The Cab etiqueta la actividad con contexto y confianza.
```

```txt
Atribución residual
```

Cuerpo:


Título:

```txt

```

Cuerpo:

```txt
Cuando un swap excede el residual de un pool, The Cab asigna el excedente entre cash-ins, inventario derivado de liquidaciones y otros estados residuales en lugar de sobre-atribuirlo.
```

Locked state — título:

```txt
Ejecutá el análisis para desbloquear governance
```

Locked state — cuerpo:

```txt
Governance requiere reconstrucción de veAERO, voter, rewards, bribes, fees y movimientos de tokens.
```

Empty state — título:

```txt
No se encontró actividad de governance
```

Empty state — cuerpo:

```txt
El análisis no detectó locks de veAERO, votos, relays o rewards de governance para esta wallet.
```

Aviso de asignación por pool:

```txt
Los rewards de governance se muestran por pool cuando los datos del protocolo permiten una asignación confiable.
```

# 9. Reglas finales para español

1. No traducir literalmente si suena artificial.
2. Mantener tono técnico, premium y cripto-nativo.
3. Usar voseo argentino/latam moderado: “Conectá”, “Ejecutá”, “Revisá”.
4. Evitar exceso de localismos.
5. Mantener términos DeFi cuando mejoran claridad: wallet, pool, rewards, governance, strategy, shares.
6. Mostrar incertidumbre con claridad: cobertura parcial, share-level, unknown.
7. No prometer rentabilidad.
8. No usar hype de trading o meme-coin.
9. No mezclar depósitos manuales con estrategias automatizadas en el lenguaje.
10. La UI debe sentirse como una cabina analítica, no como una app financiera genérica.
